Frequently Asked Questions

1. The difference between var, let, and const
Var is a functional scoped . when we access but not declare a variable before its initialized undefined. Let and Const is a block scoped. They initialized error when we access but not declare before. Var and Let are mutable and their value are reassign. Another side Const value is immutable and its value not changeable. But objects property and array value changeable by const but not reassign all object only object property values .
2. The difference between map(), forEach(), and filter()
Array map() is a one line loop magic. It makes a loop on elements and returns elements in a new array. forEach() is not returns a new array . It throws a loop and if we want output , we need to do callback function inside its. filter() method is based on condition and returns element in a new array. map() and filter() do not modify the original array . But the forEach() method modify the original array .
3. Explain arrow functions and how they are different from regular functions
Arrow function and regular function are dynamically way to write function. When we write a normal /regular function its has a arguments and write multiple line. Another side arrow function has no arguments and its returns output implicitly or explicitly. It has no callback system because it create own this. Arrow function is write short syntax.
4. How JavaScript Promises work
When we fetch a API its go to server and its give us promise. When a promise has created we can handle the result use chain .then() and .catch() method . If promise is resolved , result handle by chain .then() method. If promise is rejected result handle by .catch() method. .then() is showed a successful operation and .catch() function handle a error message.
5. How closures work in JavaScript
When we define a function inside another function, the inner function has access to the outer functions variables. And ther inner function who called a closure. Using closure, we can encapsulate private data that cannot be accessed and reassign directly from outside the closure. When outer function finished execution , the inner function still has access to outer functions variables.
